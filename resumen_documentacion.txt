# Proyecto Solar System - Documentación Completa

## Descripción General del Proyecto

Este es un **Simulador del Sistema Solar** construido en **Godot 4**. 
Muestra el Sol y 8 planetas (Mercurio, Venus, Tierra, Marte, Júpiter, Saturno, Urano y Neptuno) que 
rotan a diferentes velocidades alrededor del punto central.

## Cómo Funciona el Proyecto

### 1. **Escena Principal (main.tscn)**

La escena principal está compuesta por:

- **Fondo**: Un TextureRect que muestra la imagen de fondo del espacio
- **Sol**: El sol central, que rota lentamente
- **Planetas**: Contenedor Node2D que agrupa todos los planetas en escala 3x

Cada planeta es una instancia de la plantilla `planeta.tscn` con un Sprite2D que muestra la textura del planeta.

### 2. **Script Principal (main.gd)**

El script maneja la lógica de rotación en la función `_process()`:

```gdscript
extends Node2D

var velocidades = {
    "Mercurio": 0.1,
    "Venus": -0.05,
    "Tierra": 1.0,
    "Marte": 0.9,
    "Jupiter": 2.5,
    "Saturno": 2.3,
    "Urano": -1.5,
    "Neptuno": 1.6
}

func _process(delta):
    # Rotación del Sol
    if has_node("Sol"):
        $Sol.rotation += 0.05 * delta
    
    # Rotación de los planetas
    for planeta in $Planetas.get_children():
        if planeta.name == "Sol":
            continue
        
        var velocidad_base = velocidades.get(planeta.name, 0.5)
        planeta.rotation += velocidad_base * delta
```

### Características Clave:

| Característica | Descripción |
|---|---|
| **Diccionario de velocidades** | Almacena la velocidad de rotación para cada planeta |
| **Delta Time** | Usa `delta` para animación independiente de la velocidad de fotogramas |
| **Manejo de errores** | Verifica si los nodos existen antes de acceder a ellos |
| **Valor por defecto** | Usa 0.5 como velocidad predeterminada si el planeta no está en el diccionario |

### 3. **Plantilla de Planeta (planeta.tscn)**

Una plantilla reutilizable que contiene:

- **Planeta (Node2D)**: Contenedor principal
- **Sprite2D**: Muestra la textura del planeta
- **CollisionShape2D**: (Actualmente vacío, para futuras interacciones)

## Velocidades de Rotación

Cada planeta tiene una velocidad de rotación única definida en el diccionario `velocidades`:

| Planeta | Velocidad | Dirección | Notas |
|---|---|---|---|
| Mercurio | 0.1 | Horaria | Planeta más lento |
| Venus | -0.05 | Contrahoraria | Rota hacia atrás |
| Tierra | 1.0 | Horaria | Velocidad de referencia |
| Marte | 0.9 | Horaria | Ligeramente más lento que la Tierra |
| Júpiter | 2.5 | Horaria | Rotación más rápida |
| Saturno | 2.3 | Horaria | Rotación rápida |
| Urano | -1.5 | Contrahoraria | Rota hacia atrás |
| Neptuno | 1.6 | Horaria | Velocidad moderada-rápida |

**Nota**: Los valores negativos hacen que los planetas roten en sentido contrario a las manecillas del reloj.

## Cómo Personalizar el Proyecto

### 1. **Cambiar la Velocidad de Rotación**

Edita el diccionario `velocidades` en `main.gd`:

```gdscript
var velocidades = {
    "Tierra": 2.0,  # Cambiar de 1.0 a 2.0 para duplicar la velocidad
}
```

### 2. **Agregar un Nuevo Planeta**

1. Crea una nueva instancia de `planeta.tscn` en el contenedor Planetas
2. Agrega el archivo de textura a la carpeta assets
3. Actualiza la textura en el nodo Sprite2D del planeta
4. Agrega una entrada al diccionario `velocidades` en main.gd

### 3. **Cambiar el Fondo**

1. Reemplaza `fondo1.jpg` en la carpeta assets
2. El fondo se actualiza automáticamente en la escena

### 4. **Ajustar Posiciones de Planetas**

Edita la propiedad `position` de cada Sprite2D en `main.tscn` para cambiar las distancias orbitales.

## Detalles Técnicos

### Versión de Godot
- Construido para **Godot 4** (usa GDScript 2.0)

### Conceptos Clave Utilizados

1. **Jerarquía de Nodos**: Estructura organizada con contenedores
2. **Herencia**: Usando `extends Node2D` para funcionalidad 2D
3. **Iteración de Bucles**: Bucle `for...in` para procesar múltiples planetas
4. **Diccionarios**: Usando diccionarios para gestionar datos fácilmente
5. **Delta Time**: Animación independiente de la velocidad de fotogramas
6. **Consultas de Nodos**: `has_node()` y `get_children()` para acceso seguro de nodos

### Consideraciones de Rendimiento

- ✅ Acceso eficiente a nodos usando notación `$`
- ✅ Usa delta time para animación suave
- ✅ Cálculos mínimos por fotograma
- ✅ Escalable para más planetas

## Problemas Comunes y Soluciones

| Problema | Causa | Solución |
|---|---|---|
| El planeta no rota | Planeta no está en el diccionario `velocidades` | Agregar el nombre del planeta al diccionario con la velocidad deseada |
| Error de referencia nula | Nodo no encontrado en la escena | Verificar que la jerarquía de nodos coincida con las referencias del script |
| Velocidad incorrecta | Valor incorrecto en el diccionario | Ajustar el valor de velocidad (mayor = más rápido) |
| Planetas superpuestos | Valores de posición incorrectos | Ajustar la propiedad `position` en la escena |

## Mapa de Conexiones y Dependencias

```
main.gd (Script)
    │
    ├─→ referencias nodo "Sol"
    │   └─→ necesita Sprite2D con sol.png
    │
    ├─→ referencias contenedor "Planetas"
    │   └─→ contiene 8 nodos hijo de planeta:
    │       ├─→ "Mercurio" → necesita mercurio.png
    │       ├─→ "Venus" → necesita venus.png
    │       ├─→ "Tierra" → necesita tierra.png
    │       ├─→ "Marte" → necesita marte.png
    │       ├─→ "Jupiter" → necesita jupiter.png
    │       ├─→ "Saturno" → necesita saturno.png
    │       ├─→ "Urano" → necesita urano.png
    │       └─→ "Neptuno" → necesita neptuno.png
    │
    └─→ diccionario velocidades
        └─→ coincide exactamente con nombres de nodos
            (Si los nombres no coinciden, usa 0.5 por defecto)

main.tscn (Escena)
    │
    ├─→ referencias main.gd
    ├─→ referencias fondo1.jpg
    ├─→ referencias planeta.tscn (9 instancias)
    └─→ referencias todas las texturas de planetas


DEPENDENCIAS CRÍTICAS ⚠️:
  ✓ Nombres de nodos en escena = Claves en diccionario velocidades
  ✓ main.gd adjunto al nodo Main
  ✓ Todos los archivos de textura presentes en assets/
  ✓ planeta.tscn existe para instancias
```
